1. 프로젝트 개요
목적: 생성형 AI(또는 MCP 서버)에서 사용할 신문사 교열·편집 지침을 JSON 형식으로 변환

범위:

기자별 교열 정보 (초기 파트)

편집·레이아웃(디자인) 지침

문장부호·약물·배치 규정

기사 유형별 제목 작성법

마지막으로 “자주 틀리는 말 + 복수 표준어” 등 한국어 문법·어휘 교정 지침

방식: 각 텍스트 파일의 내용을 (1) 요약하고, (2) 본문을 섹션·하위 섹션으로 구분하여 JSON 객체로 구조화.

2. 작업 파일 요약
아래는 크게 **“MCP 서버 컨텍스트에 맞게 요약/정리”**된 주요 파일 및 그 JSON 변환 작업의 흐름입니다.

기본 원칙 / 리드 쓰기 / 문장 요건 / 피해야 할 표현들

예: “1기사 1주제”, “1문장 1정보”, “평이성과 명료성” 등 문장 작성법

“리드 문장은 기자 역량의 핵심”이나 “직접 vs 간접 리드” 같은 실무 팁

“금칙어(사회 통념상 문제될 표현) / 외래어 사용 방식” 등

기사 유형별 작성 요령

스트레이트, 해설, 화제성, 숫자성, 인터뷰, 르포 기사 등의 제목·문체 특징

“숫자를 단순 나열보다 2배·10배로 치환” 등 실무 노하우

레이아웃(신문 편집) 지침

“균형(Balance), 리듬(Rhythm), 대비(Contrast), 통일성(Unification)” 같은 디자인 원칙

“가로쓰기 편집은 모듈러 디자인” “박스 기사 연달아 붙이지 않기” 등 편집 실무 팁

“역S자형 눈 흐름 고려” “메인 시각물은 중앙 윗부분 배치” 등 시각적 고려사항

제목달기

“첫 줄에서 정곡을 찔러라” “동사적 의미, 능동형 사용” “피동 자제”

“독자 입장 고려” “감각적·리듬감 있게” “확정되지 않은 사실은 단정적 표현 삼가” 등

자주 틀리는 말 / 복수 표준어 (마지막 제공된 긴 파일)

“때문”은 의존명사, “에/에게” 구분, 사이시옷 불규칙 규정, …

“방증 vs. 반증”, “과반수 vs. 과반수 이상”, “뒤처지다 vs. 뒤쳐지다”, “엄한 vs. 애먼” 등

국립국어원에서 새로 인정한 복수 표준어 (간질이다↔간지럽히다 등)

3. JSON 변환 원칙
모든 텍스트를 단순히 (a) 한꺼번에 담기보다는,

file_id: 각 문서(파일)를 구분할 유일 식별자

title: 문서 제목

tags: 검색·분류용 키워드(“레이아웃”, “문체”, “제목달기” 등)

content: 원문 또는 요약본

sections (배열):

heading: 섹션 제목(“1. 균형”, “2. 리듬” 등)

content: 그 항목의 주요 내용 or 예시

필요시 subSections, table, examplesTable 등을 사용 (e.g., “틀린 문장 vs. 바른 문장” 표)

이를 통해 AI / MCP 서버에서 **특정 항목(예: “리듬(Rhythm) 편집 원칙”)**만 검색하거나, 전체 문서를 한 번에 불러올 수 있음.

4. 인수인계 포인트
파일별 JSON: 각 문서를 독립된 JSON으로 만들어둠 → DB/서버에 저장 시 file_id로 구분

통합 용이: MCP 서버(또는 AI 컨텍스트)에서 “파일별 섹션” 검색할 수 있도록 구조화

확장: 내용이 늘어나면, sections 내 subSections나 table 항목만 추가 → 기존 구조 깔끔 유지

주요 지침:

“가장 작은 단위(섹션)로 쪼개고, 표가 필요한 경우 table”

“중립적 표현” “피동 표현 자제” 등 공통 원칙은 여러 문서에서 중복 가능 (각 문서별 섹션에 존재) → 실무 시 중복 업데이트 주의.

5. 앞으로의 활용
초기 시범: “문체 교정” or “레이아웃 가이드” 등 AI에게 부가 정보로 제공 → “이 문단에서 ‘뒤쳐지다’를 찾고 수정해줘” 식의 기능

기자 교육: 각 기자별로 JSON 문서를 에디터/ChatGPT 등에 붙여, “교열 규칙 확인” “제목달기 기본” 등 확인

확장:

새 항목(용례/금칙어/기타 편집 팁)을 sections나 table에 추가

기존 JSON 형식을 깨지 않게, 새 subsections만 append

6. 요약
결과:

(a) 수십 개의 교열·편집·레이아웃·문체 지침을 JSON 구조로 변환

(b) 각 문서마다 file_id, title, sections 등으로 구성

(c) “자주 틀리는 말”/“복수 표준어” 등 큰 문서는 표·배열 형태로 처리

인수인계:

다음 작업자는 이 JSON 구조(파일별 file_id)에 새 규칙/표/섹션을 추가 가능

AI/MCP 쪽에서 “특정 섹션만 검색 or 전체 컨텍스트” 식으로 유연하게 활용 가능
